# 🌟 About H Book

H Book is a complete **Payroll & Employee Management System** designed to help
organizations digitize and simplify their HR operations. It combines the flexibility
of **Google Apps Script** with the power of **Google Sheets** to create a lightweight,
secure, and scalable web application that requires no dedicated server.

Unlike traditional desktop payroll software, H Book runs directly in the browser,
allowing authorized users to access employee information, attendance records,
salary details, and reports from anywhere with appropriate permissions.

The project is designed with a modern responsive interface, making it equally
comfortable on desktops, tablets, and mobile devices.

---

# 💡 Why H Book?

Managing employees manually becomes increasingly difficult as an organization grows.
Payroll calculations, attendance tracking, compliance updates, salary records,
and employee documents often end up scattered across multiple spreadsheets.

H Book brings all these workflows together into a single application.

It enables administrators to:

- Maintain a centralized employee database.
- Record daily attendance efficiently.
- Process monthly payroll accurately.
- Generate professional salary slips.
- Store employee documents in Google Drive.
- Manage users securely through role-based access.
- Access data from anywhere using Google Workspace.

The system focuses on reliability, ease of use, and automation while minimizing
manual work and reducing the possibility of payroll errors.

# ⚙️ Backend Architecture

The backend of **H Book** is developed entirely using **Google Apps Script (GAS)**. It acts as the application's server-side engine, handling authentication, business logic, data validation, communication with Google Sheets, Google Drive integration, and all CRUD (Create, Read, Update, Delete) operations.

Unlike traditional web applications that require dedicated servers and databases, H Book leverages Google Workspace as its cloud infrastructure, making deployment simple, secure, and cost-effective.

---

# 🏛 Backend Philosophy

The backend has been designed around five core principles:

- 🔒 Security First
- ⚡ High Performance
- 📦 Modular Architecture
- ☁ Native Google Workspace Integration
- 🔄 Reusable Business Logic

Every backend function is responsible for a specific task and follows a structured workflow to ensure maintainability and scalability.

---

# 🗄️ Database Architecture

The **H Book Database Layer** is built entirely on **Google Sheets**, transforming spreadsheets into a structured cloud-based database. Every module in the application communicates with dedicated worksheets through Google Apps Script, providing centralized storage, simplified maintenance, and seamless integration with the Google Workspace ecosystem.

Unlike traditional SQL databases, H Book uses a modular spreadsheet architecture where each worksheet represents a logical entity such as employees, users, attendance, payroll, or application settings.
